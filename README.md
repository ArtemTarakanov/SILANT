# Силант - Система учета техники

Веб-приложение для управления складской техникой, ведения истории технического обслуживания и регистрации рекламаций.

## Описание

Проект разработан для компании "Силант" и предназначен для автоматизации учета погрузчиков, их технического обслуживания и ремонтов. Система поддерживает работу трех типов пользователей с разными уровнями доступа.

## Основные возможности

- Учет машин с полной технической информацией
- Ведение журнала технического обслуживания
- Регистрация и отслеживание рекламаций
- Поиск техники по заводскому номеру
- Система справочников для стандартизации данных
- REST API для интеграции с внешними системами
- Разграничение прав доступа по ролям

## Роли пользователей

### Гость
- Поиск машины по заводскому номеру
- Просмотр базовой информации о технике

### Клиент
- Просмотр своих машин
- Добавление записей о ТО
- Просмотр рекламаций

### Сервисная организация
- Просмотр обслуживаемых машин
- Добавление записей о ТО
- Создание рекламаций

### Менеджер
- Полный доступ ко всем данным
- Управление справочниками
- Создание и редактирование машин

## Установка

1. Клонируйте репозиторий:
```bash
git clone <repository-url>
cd silant
```

2. Создайте виртуальное окружение:
```bash
python -m venv venv
source venv/bin/activate  # для Linux/Mac
venv\Scripts\activate     # для Windows
```

3. Установите зависимости:
```bash
pip install -r requirements.txt
```

4. Создайте файл `.env` на основе `.env.example`:
```bash
cp .env.example .env
```

5. Настройте переменные окружения в `.env`:
```
DJANGO_SECRET_KEY=your-secret-key
DEBUG=True
DATABASE_NAME=silant_db
DATABASE_USER=postgres
DATABASE_PASSWORD=your-password
DATABASE_HOST=localhost
DATABASE_PORT=5432
DJANGO_ALLOWED_HOSTS=localhost,127.0.0.1
```

6. Примените миграции:
```bash
python manage.py migrate
```

7. Создайте суперпользователя:
```bash
python manage.py createsuperuser
```

8. Загрузите тестовые данные (опционально):
```bash
python manage.py loaddata fixtures/initial_data.json
```

9. Запустите сервер:
```bash
python manage.py runserver
```

Приложение будет доступно по адресу: http://localhost:8000

## Тестовые пользователи

После загрузки тестовых данных доступны следующие учетные записи:

- **Менеджер**: `manager` / `password123`
- **Клиент**: `client` / `password123`
- **Сервисная организация**: `service` / `password123`

## API

Документация API доступна по адресам:
- Swagger: http://localhost:8000/swagger/
- ReDoc: http://localhost:8000/redoc/

### Основные эндпоинты

- `GET /api/machines/` - список машин
- `GET /api/maintenance/` - список ТО
- `GET /api/claims/` - список рекламаций
- `GET /api/search/?factory_number=XXX` - поиск машины

## Структура проекта

```
├── claims/           # Модуль рекламаций

├── static/           # Статические файлы

```
